    REST API

    • 資產劃轉API發布

    鑒權/認證

    請先生成測試網 或 主網API KEY。
    REST接口根URL:

    • 測試網:
      https://api-testnet.bybit.com

    • 主網(請根據您的網絡情況選擇以下任意一個地址使用):
      https://api.bybit.com
      https://api.bytick.com

    所有private接口都需要認證，public 接口不需要認證。

    公共參數

    需要簽名的接口必須包含以下參數:

    • api_key
    • timestamp - UTC毫秒時間戳
    • sign - 請求參數簽名

    另外我們提供可選的recv_window參數（單位是毫秒，默認值為5000），來指定請求在多長時間內有效，同時用來防止重放攻擊。

    註意，timestamp參數必須符合如下規則: server_time - recv_window <= timestamp < server_time + 1000，其中server_time是Bybit服務器時間，您可以通過服務器時間接口獲取。

    構建請求

    拼接參數示例（以修改槓桿率為例）：

    param_str = "api_key=B2Rou0PLPpGqcU0Vu2&leverage=100&symbol=BTCUSDH21&timestamp=1542434791747"

# api_key=B2Rou0PLPpGqcU0Vu2&
# leverage=100&
# symbol=BTCUSDH21&
# timestamp=1542434791747

    參數按照 字母順序 排列，然後計算sign。

    1. 所有參數按照字母順序排序，然後按照query string格式拼接。

    2. 使用HMAC_SHA256算法對第1步中拼接的query string簽名，並轉換為16進製字符串，得出sign參數。

    GET請求格式（查詢賬號余額）: 

    GET /asset/v1/private/transfer/list?api_key=q1ksyOX2T0G2SkK8nu&recvWindow=5000&timestamp=1623208423972&sign=b452640c21a2c9eaec30d24a9bce1a9660d1fb9d07ccc0d623a2a4fca0940095 HTTP/1.1
Host: api-testnet.bybit.com

    POST請求格式（創建訂單）: 

    POST /asset/v1/private/transfer HTTP/1.1
Host: api-testnet.bybit.com
Content-Type: application/json

{
    "from_account_type": "SPOT",
    "to_account_type": "CONTRACT",
    "amount": "0.01",
    "coin": "USDT",
    "transfer_id": "11ff9b44-2d5d-4293-913d-4597c9ad2170",
    "sign": "{{signature}}",
    "timestamp": "{{timestamp}}",
    "api_key": "{{bybit-api-key}}",
    "recv_window": "50000"
}

    觸發劃轉動作

    請註意該接口入參請求數據中的transfer_id字段為UUID全局唯一。

    請註意該接口支持期貨賬戶、現貨賬戶、理財賬戶。

    轉入和轉出賬戶類型不能相同。

    HTTP 請求

    POST /asset/v1/private/transfer

    請求參數

    參數 是否必須 類型 說明
    transfer_id true string UUID,全局唯一
    coin true string 幣種類型
    amount true string 兌入數量
    from_account_type true string 賬戶類型
    to_account_type true string 賬戶類型

    返回參數

    參數 類型 說明
    transfer_id string UUID,全局唯一

    支持的充值列表查詢接口

    請求示例

    curl --location --request GET 'https://api-testnet.bybit.com/asset/v1/public/deposit/allowed-deposit-list?coin=ETH&chain=ETH&page_index=1&page_size=10'

    響應示例

    "ret_msg": "OK",
    "ext_code": "",
    "result": {
        "config_list": [
            {
                "coin": "ETH",
                "chain": "ETH",
                "coin_show_name": "ETH",
                "chain_type": "ETH",
                "block_confirm_number": 12,
                "min_deposit_amount": "0"
            }
        ],
        "total": "1"
    },
    "ext_info": null,
    "time_now": 1652841868446
}

    是一個公共接口。 coin和chain必須在輸入參數中配對出現。 否則，視為無條件查詢。

    HTTP 請求

    GET /asset/v1/public/deposit/allowed-deposit-list

    請求參數

    參數 是否必須 類型 說明
    coin false string 充值幣種. coin和chain必須在輸入參數中配對出現。 否則，視為無條件查詢。
    chain false string 充值鏈名. coin和chain必須在輸入參數中配對出現。 否則，視為無條件查詢。
    page_index false long 頁面索引：用於分頁。 默認值：1
    page_size false long 每頁的數量。 默認值：10

    返回參數

    參數 類型 說明
    config_list Array Object
    coin string 充值的幣種
    chain string 鏈名
    coin_show_name string Coin show name
    chain_type string 鏈名
    block_confirm_number long 充值上賬確認數
    min_deposit_amount string 最低充值金額

    規則：只查詢現貨賬戶的充值記錄，按id倒序進行，開始時間和截止時間差最大限制為30天

    HTTP 請求

    GET /asset/v1/private/deposit/record/query

    請求參數

    參數 是否必須 類型 說明
    start_time false long 開始時間 (精確到秒)。 默認為當前時間之前30天
    end_time false long 結束時間 (精確到秒)。 默認為當前時間
    coin false string 幣種名：舉例，BTC。默認全部
    cursor false string 游標信息：用來分頁。 默認空
    direction false aaa 上一頁Prev，下一頁 Next。默認下一頁
    limit false long 每頁條數。0< limit <=50 默認為50
    timestamp true long 時間戳

    返回參數

    參數 類型 說明
    coin string 充值的幣種
    chain string 鏈名
    amount string 充值的數量
    tx_id string 交易ID。充值失敗/取消充值：為空
    status int

    0=未知
    1=待確認
    2=上賬中
    3=已完成
    4=充值失敗

    to_address string 充值的目標地址
    tag string 充值目標地址的tag
    deposit_fee string 充值手續費
    success_at int64 最後更新時間
    confirmations string 确认区块的数量
    tx_index string 交易序列号
    block_hash string 鏈上的哈希數
    cursor string 游標信息：用來分頁

    頻率限製

    IP頻率限製

    The IP limits are shared across all endpoints: futures, spot, and account asset.

    Bybit has different IP frequency limits for GET and POST method:

    • GET & DELETE method:
      • 12 requests per second continuously for 2 minutes
      • 18 requests per second continuously for 5 seconds
    • POST method:
      • 10 requests per second continuously for 2 minutes
      • 15 requests per second continuously for 5 seconds

    充值與提現接口頻率限製和現貨劃轉相關接口頻率限製是分開計算的。

    現貨劃轉接口的頻率限製

    頻率限製 請求路徑 消耗
    20/min /asset/v1/private/transfer 1 / request
    20/min /asset/v1/private/sub-member/transfer 1 / request
    60/min /asset/v1/private/sub-member/member-ids 1 / request
    60/min /asset/v1/private/transfer/list 1 / request
    60/min /asset/v1/private/sub-member/transfer/list 1 / request
