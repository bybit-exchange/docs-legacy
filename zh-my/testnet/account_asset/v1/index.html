    鑒權/認證

    請先生成測試網 或 主網API KEY。
    REST接口根URL:

    • 測試網:
      https://api-testnet.bybit.com

    • 主網(請根據您的網絡情況選擇以下任意一個地址使用):
      https://api.bybit.com
      https://api.bytick.com

    所有private接口都需要認證，public 接口不需要認證。

    公共參數

    需要簽名的接口必須包含以下參數:

    • api_key
    • timestamp - UTC毫秒時間戳
    • sign - 請求參數簽名

    另外我們提供可選的recv_window參數（單位是毫秒，默認值為5000），來指定請求在多長時間內有效，同時用來防止重放攻擊。

    註意，timestamp參數必須符合如下規則: server_time - recv_window <= timestamp < server_time + 1000，其中server_time是Bybit服務器時間，您可以通過服務器時間接口獲取。

    構建請求

    拼接參數示例（以修改槓桿率為例）：

    param_str = "api_key=XXXXXXXXXXX&leverage=100&symbol=BTCUSDH21&timestamp=1542434791747"

# api_key=XXXXXXXXXXX&
# leverage=100&
# symbol=BTCUSDH21&
# timestamp=1542434791747
    為了幫助診斷高級網絡問題，您可以考慮將 cdn-request-id 添加到您的請求標頭中。對於每個請求，它的值應該是唯一的。

    參數按照 字母順序 排列，然後計算sign。

    1. 所有參數按照字母順序排序，然後按照query string格式拼接。

    2. 使用HMAC_SHA256算法對第1步中拼接的query string簽名，並轉換為16進製字符串，得出sign參數。

    GET請求格式（查詢賬號余額）: 

    GET /asset/v1/private/transfer/list?api_key=XXXXXXXXXXX&recvWindow=5000&timestamp=1623208423972&sign=b452640c21a2c9eaec30d24a9bce1a9660d1fb9d07ccc0d623a2a4fca0940095 HTTP/1.1
Host: api-testnet.bybit.com

    POST請求格式（創建訂單）: 

    POST /asset/v1/private/transfer HTTP/1.1
Host: api-testnet.bybit.com
Content-Type: application/json

{
    "from_account_type": "SPOT",
    "to_account_type": "CONTRACT",
    "amount": "0.01",
    "coin": "USDT",
    "transfer_id": "11ff9b44-2d5d-4293-913d-4597c9ad2170",
    "sign": "{{signature}}",
    "timestamp": "{{timestamp}}",
    "api_key": "{{bybit-api-key}}",
    "recv_window": "50000"
}

    觸發劃轉動作

    要發送賬戶類型和父子不可知的轉賬，請使用創建萬能轉賬 接口。

    請註意該接口入參請求數據中的transfer_id字段為UUID全局唯一。

    請註意該接口支持期貨賬戶、現貨賬戶、理財賬戶。

    轉入和轉出賬戶類型不能相同。

    升級到統一保證金帳戶後，僅支持從現貨帳戶轉帳USDT到統保帳戶。

    HTTP 請求

    POST /asset/v1/private/transfer

    請求參數

    參數 是否必須 類型 說明
    transfer_id true string UUID,全局唯一
    coin true string 幣種類型
    amount true string 兌入數量
    from_account_type true string 賬戶類型
    to_account_type true string 賬戶類型

    返回參數

    參數 類型 說明
    transfer_id string UUID,全局唯一

    觸發子母劃轉動作

    要發送賬戶類型和父子不可知的轉賬，請使用創建萬能轉賬 接口。

    請註意該接口入參請求數據中的transfer_id字段為UUID全局唯一。

    該接口只能用於母子賬號之間的劃轉。

    一個子賬號只會歸屬給一個母賬號。

    HTTP 請求

    POST /asset/v1/private/sub-member/transfer

    請求參數

    參數 是否必須 類型 說明

    返回參數

    參數 類型 說明

    頻率限製

    有兩種類型的頻率限製: IP 頻率限製 (每秒计算) and API 頻率限製 (每分鐘计算). IP 頻率限制將計算您的所有請求，而 API 頻率限制僅計算您經過身份驗證的請求。

    IP頻率限製

    如果您收到 HTTP 403（拒絕訪問）響應, 您的 IP 已被暫時或永久禁止。 您應立即查看以下指南，以確保您的應用程序不會繼續違反限制。 如果您在 30 分鐘後仍被禁止，則您可能會被永久禁止。

    Bybit根據請求方式有不同的IP頻率限制。 我們不建議您在這些限制的邊緣運行您的應用程序，以防異常的網絡活動導致意外違規。

    • GET, POST & DELETE 方法 (共享):
      • 連續 5 秒鐘內每秒不超過 120 個請求

    所有到api.bybit.com或api.bytick.com的流量共享此限制，無論是訪問現貨、期貨還是期權。

    違反限制後，您的 IP 將被禁止一段時間（通常為 30 分鐘）。 持續違反限制將導致永久禁止。 我們不能撤銷永久禁令或縮短臨時禁令。
